    Description of Project

    Starting with a semi-specific user goal, students will research, design, and test a mobile app that has a clear primary function.

    • This is an iterative process, so there will be an expectation that goals and solutions will change as new ideas are explored and as students test and get feedback
    • Projects can include design aspects that exist outside of the mobile app (eg. email notifications, real-world locations, tickets, websites, etc.)
    • There should be a single primary goal, but the path that gets users there should include at least one or two meaningful choices
    • Secondary goals are allowed, but not if they get in the way of the primary goal
